# Requirements: artpqUX 퍼블리싱 가이드 시스템

**Defined:** 2026-03-26
**Core Value:** 신규 프로젝트 시작 시 퍼블리싱 규칙을 처음부터 다시 정하지 않고, 검증된 팀 표준을 즉시 적용할 수 있어야 한다.

## v1 Requirements

### SCSS 구조 (SCSS)

- [ ] **SCSS-01**: Bootstrap 5 SCSS를 `@use`로 임포트하는 표준 진입점 파일(`style.scss`)이 존재한다
- [ ] **SCSS-02**: Bootstrap 변수 오버라이드 파일(`_variables.scss`)이 Bootstrap 로드 이전에 적용된다
- [ ] **SCSS-03**: 팀 공통 믹스인 파일(`_mixin.scss`)이 표준화된다 (focus, flex, ellipsis, position, drop-shadow, ani 등)
- [ ] **SCSS-04**: 공통 기본 스타일 파일(`_common.scss`)이 표준화된다 (reset 보완, sr-only, 기본 요소 스타일)
- [ ] **SCSS-05**: 폰트 파일(`_font.scss`)이 표준화된다 (Pretendard GOV / Noto Sans KR 전환 가이드 포함)
- [ ] **SCSS-06**: CSS Custom Properties 루트 파일(`_root.scss`)에 팀 공통 토큰이 정의된다
- [ ] **SCSS-07**: SCSS 파일 로드 순서가 문서화된다 (variables → mixin → font → common → root → components)

### 디자인 토큰 (TOKEN)

- [ ] **TOKEN-01**: 색상 토큰이 CSS Custom Properties로 정의된다 (`--color-primary`, `--color-text`, `--color-border` 등)
- [ ] **TOKEN-02**: 타이포그래피 토큰이 정의된다 (`--font-size-base`, `--font-weight-*`, `--leading-*`)
- [ ] **TOKEN-03**: 간격 토큰이 4px 기반 스케일로 정의된다 (`--spacing-xs` ~ `--spacing-3xl`)
- [ ] **TOKEN-04**: 그림자/전환/z-index 토큰이 정의된다
- [ ] **TOKEN-05**: Bootstrap `$primary` 등 주요 변수와 CSS Custom Properties의 매핑 관계가 문서화된다
- [ ] **TOKEN-06**: 프로젝트별 색상 오버라이드 방법이 문서화된다 (`_project-overrides.scss` 패턴)

### 반응형 (RESP)

- [x] **RESP-01**: 팀 표준 브레이크포인트 4단계가 정의된다 (모바일 ~767px / 태블릿 768~1023px / 소형PC 1024~1279px / PC 1280px~)
- [x] **RESP-02**: 반응형 미디어쿼리 믹스인이 제공된다 (`respond-to(mobile)`, `respond-to(tablet)` 등)
- [x] **RESP-03**: 모바일 퍼스트 작성 기준이 문서화된다

### 컨벤션 (CONV)

- [x] **CONV-01**: 클래스 네이밍 규칙이 문서화된다 (Bootstrap 유틸리티 사용 기준, 커스텀 클래스 작성 규칙)
- [x] **CONV-02**: SCSS 작성 규칙이 문서화된다 (중첩 깊이 제한, 속성 순서, 주석 규칙)
- [x] **CONV-03**: HTML 마크업 기본 구조가 문서화된다 (lang, charset, viewport, 본문건너뛰기 등)
- [x] **CONV-04**: 코딩 스타일 규칙이 문서화된다 (2 spaces, 한국어 주석, 파일 인코딩)
- [x] **CONV-05**: Git 커밋 메시지 규칙이 문서화된다 (한국어, 명령형, 타입 분류)

### 접근성 (A11Y)

- [x] **A11Y-01**: 본문 건너뛰기 링크 패턴이 제공된다 (`skip-to-content`)
- [x] **A11Y-02**: 스크린 리더 전용 텍스트 패턴이 제공된다 (`.sr-only`, `.sr-only-focusable`)
- [x] **A11Y-03**: 포커스 스타일 표준이 정의된다 (focus 믹스인, outline 기준)
- [x] **A11Y-04**: 이미지 대체텍스트 기준이 문서화된다 (장식 이미지 `alt=""`, 의미 이미지 설명)
- [x] **A11Y-05**: 폼 접근성 패턴이 문서화된다 (label-input 연결, error 메시지, required 표시)
- [x] **A11Y-06**: 색상 대비 기준이 문서화된다 (텍스트 4.5:1, 대형 텍스트 3:1)
- [x] **A11Y-07**: 키보드 탐색 지원 기준이 문서화된다 (탭 순서, 포커스 트랩, 단축키)
- [x] **A11Y-08**: KRDS 기반 접근성 체크리스트가 제공된다 (공공기관 납품 체크리스트)
- [x] **A11Y-09**: 동적 콘텐츠 접근성 패턴이 제공된다 (aria-live, role="alert", 모달 포커스 트랩)

### 컴포넌트 (COMP)

- [x] **COMP-01**: 헤더/GNB 마크업 패턴이 제공된다 (PC 메뉴, 모바일 전체메뉴, 접근성 포함)
- [x] **COMP-02**: 버튼 컴포넌트 패턴이 제공된다 (Bootstrap 확장, variant, 상태별 스타일)
- [x] **COMP-03**: 폼 컴포넌트 패턴이 제공된다 (input, select, textarea, checkbox, radio — 접근성 포함)
- [x] **COMP-04**: 카드 컴포넌트 패턴이 제공된다
- [x] **COMP-05**: 테이블 컴포넌트 패턴이 제공된다 (caption, scope, 접근성 포함)
- [x] **COMP-06**: 모달 컴포넌트 패턴이 제공된다 (포커스 트랩, aria-modal, 닫기 버튼)
- [x] **COMP-07**: 탭 컴포넌트 패턴이 제공된다 (role="tablist", aria-selected)
- [x] **COMP-08**: 페이지네이션 컴포넌트 패턴이 제공된다 (aria-label, aria-current="page")
- [x] **COMP-09**: 브레드크럼 컴포넌트 패턴이 제공된다 (nav, aria-label="breadcrumb")
- [x] **COMP-10**: 슬라이더/Swiper 접근성 패턴이 제공된다 (aria-live, role, 정지 버튼)

### 문서 사이트 (DOCS)

- [x] **DOCS-01**: Eleventy 기반 문서 사이트가 로컬에서 실행된다 (`npm run serve`)
- [x] **DOCS-02**: 컨벤션 섹션이 문서화된다 (SCSS 규칙, 네이밍, 코딩 스타일)
- [x] **DOCS-03**: 토큰 섹션이 문서화된다 (색상, 타이포, 간격 토큰 목록)
- [x] **DOCS-04**: 컴포넌트 섹션이 문서화된다 (각 컴포넌트 HTML 스니펫 + 접근성 주의사항)
- [x] **DOCS-05**: 접근성 섹션이 문서화된다 (체크리스트, 패턴, 색상 대비 가이드)
- [x] **DOCS-06**: 문서 사이트에서 코드 복사 기능이 동작한다
- [x] **DOCS-07**: 문서 사이트가 npm 단일 명령으로 빌드된다 (`npm run build`)

### 스타터 킷 (STARTER)

- [x] **STARTER-01**: `starter/` 디렉토리가 신규 프로젝트 복사본으로 사용 가능하다
- [x] **STARTER-02**: 스타터 킷에 표준 SCSS 파일 구조가 포함된다 (variables, mixin, common, font, root)
- [x] **STARTER-03**: 스타터 킷에 기본 HTML 보일러플레이트가 포함된다 (본문건너뛰기, 메타태그 등)
- [x] **STARTER-04**: 스타터 킷 README에 프로젝트 시작 방법이 문서화된다

## v2 Requirements

### 자동화 (AUTO)

- **AUTO-01**: Stylelint로 SCSS 네이밍 규칙을 자동 검증한다
- **AUTO-02**: pa11y-ci로 접근성 기준을 자동 테스트한다
- **AUTO-03**: npm 빌드 파이프라인에 lint + a11y 테스트가 통합된다

### 확장 문서 (EXT)

- **EXT-01**: 다크모드 대응 가이드가 제공된다
- **EXT-02**: 인쇄 스타일 가이드가 제공된다
- **EXT-03**: 애니메이션/전환효과 가이드가 제공된다

## Out of Scope

| Feature | Reason |
|---------|--------|
| 피그마 디자인 가이드 | 퍼블리싱 전담팀, 디자인 도구 규칙은 범위 외 |
| React/Vue 컴포넌트 | HTML/CSS 퍼블리싱 팀 특성상 불필요 |
| Bootstrap 완전 제거 | 현행 Bootstrap 5 기반 유지 |
| IE11 지원 | 2026년 기준 공공기관도 IE11 지원 종료 |
