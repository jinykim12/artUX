**1-1. `scss/4-elements/_common.scss` 신규 생성 (per SCSS-04)**

Bootstrap Reboot 이후 팀 보완 초기화 레이어. Reboot이 이미 처리하는 항목(`* { box-sizing: border-box }`, `img { max-width: 100% }` 등)은 중복 작성하지 않는다.

절대로 `outline: none`을 작성하지 않는다 — 공공기관 납품 접근성 위반(KWCAG 2.1 AA). 포커스 스타일 재정의는 Phase 5(A11Y-03)에서 처리.

```scss
// ====================================================
// artpqUX 공통 기본 스타일 — Bootstrap Reboot 이후 보완
//
// Bootstrap Reboot이 이미 처리하는 항목은 중복 작성하지 않는다:
//   - * { box-sizing: border-box }
//   - body 기본 font-family, font-size, line-height, color
//   - p { margin-top: 0 }
//   - img { max-width: 100% }
//
// [접근성] outline: none 사용 금지
//   키보드 포커스 인디케이터 제거는 KWCAG 2.1 AA 위반.
//   포커스 스타일 재정의는 Phase 5에서 focus() 믹스인으로 처리.
// ====================================================

// [한국어 줄바꿈 — 공공기관 필수]
// Bootstrap Reboot은 word-break을 설정하지 않음
html,
body {
  word-break: keep-all;   // 한국어 단어 단위 줄바꿈
  word-wrap: break-word;  // 긴 URL 등 강제 줄바꿈
}

// [기본 링크 스타일]
// Bootstrap 5 Reboot: color: $link-color + text-decoration: underline
// 팀 관행: underline 제거
a {
  text-decoration: none;

  &:hover {
    text-decoration: none;
  }
}

// [버튼 초기화]
// outline: none은 접근성 위반 — 사용 금지
// 포커스 스타일은 Phase 5에서 focus() 믹스인으로 별도 정의
button {
  border-radius: 0;
  white-space: nowrap;
}

// [비활성화 상태]
button:disabled,
button[disabled],
.disabled {
  opacity: 0.5;
  cursor: not-allowed;
}

// [스크린리더 전용 텍스트 — A11Y-02 기반]
// Bootstrap 5에도 .visually-hidden이 있으나 팀 관행명(.sr-only) 유지
.sr-only {
  position: absolute !important;
  overflow: hidden !important;
  width: 1px !important;
  height: 1px !important;
  padding: 0 !important;
  margin: -1px !important;
  clip: rect(0, 0, 0, 0) !important;
  border: 0 !important;
}

.sr-only-focusable:active,
.sr-only-focusable:focus {
  position: static !important;
  width: auto !important;
  height: auto !important;
  margin: 0 !important;
  overflow: visible !important;
  clip: auto !important;
}
```

**2-1. `scss/style.scss` 업데이트 (per SCSS-07)**

Plan 01에서 `3-generic/_index.scss`에 `@forward 'root'`가 추가되었으므로, `style.scss`에 `@use '3-generic' as generic`을 추가한다. 기존 `@use` 레이어 순서에 3-generic을 삽입하되, `@import '3-generic/vendor'`는 그대로 마지막에 유지한다 (Sass @use before @import 규칙).

상단 주석 블록도 5종 파일 반영으로 업데이트한다.

```scss
// ====================================================
// artpqUX 퍼블리싱 가이드 — 메인 진입점
//
// 로드 순서 (ITCSS 기반):
//   1-settings  → 변수 (문서화 원본) — CSS 출력 없음
//   2-tools     → 믹스인 6종 — CSS 출력 없음
//   3-generic   → _root.scss (CSS Custom Properties 뼈대)
//   4-elements  → _base.scss (62.5% REM), _font.scss, _common.scss
//   5-objects   → 레이아웃 패턴
//   6-components → UI 컴포넌트
//   7-utilities → 유틸리티 클래스
//   Bootstrap   → @import 격리 (@use 이후 마지막 배치)
//
// Bootstrap 임포트 전략:
//   Bootstrap 5.3.x는 내부적으로 @import 사용.
//   3-generic/_vendor.scss에 @import로 격리하고,
//   나머지 레이어는 @use/@forward 기반으로 유지한다.
//   Sass 규칙상 @use는 @import보다 먼저 선언해야 하므로,
//   @use 계층을 모두 먼저 선언하고 @import를 그 뒤에 배치한다.
//   $font-size-base 등 Bootstrap 오버라이드 변수는
//   _vendor.scss 내부에서 @import 스코프로 선언한다.
//
// Bootstrap 오버라이드 변수 위치:
//   3-generic/_vendor.scss 내 @import 스코프에서 선언
//   1-settings/_variables.scss는 문서화 원본 역할만 수행
// ====================================================

// @use 계층 — Sass 규칙상 @import보다 먼저 위치해야 한다
// 1. 설정 (변수 문서화 원본)
@use '1-settings' as settings;

// 2. 도구 (믹스인 6종)
@use '2-tools' as tools;

// 3. Generic (CSS Custom Properties 뼈대)
@use '3-generic' as generic;

// 4~7. 커스텀 스타일 레이어
@use '4-elements' as elements;
@use '5-objects' as objects;
@use '6-components' as components;
@use '7-utilities' as utilities;

// Bootstrap (@import 격리 — @use 이후에 위치)
// _vendor.scss 내부가 @import 기반이므로 @use 불가
// CSS 출력 순서: Bootstrap이 커스텀 레이어 뒤에 오지만,
// 클래스 선택자 기반이므로 요소 스타일과 충돌 없음
@import '3-generic/vendor';
```
