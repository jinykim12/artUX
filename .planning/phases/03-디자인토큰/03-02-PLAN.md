```scss
// ====================================================
// Bootstrap 5.3 SCSS 격리 임포트
// Bootstrap 내부는 @import 기반 — Dart Sass deprecation 경고 발생은 정상
// 이 파일을 통해 Bootstrap @import를 격리하고,
// 나머지 프로젝트 SCSS는 순수 @use/@forward 기반으로 유지한다
// Bootstrap 6.x에서 @use 마이그레이션 완료 시 이 파일을 교체할 것
// ====================================================

// Bootstrap 오버라이드 변수 — @import 스코프에서 선언해야 Bootstrap이 인식한다
// (Sass @use 모듈 시스템은 네임스페이스로 격리되므로 Bootstrap이 못 읽음)
//
// ── 토큰 매핑 주석 (TOKEN-05) ──────────────────────
// [중요] Bootstrap Sass 변수는 컴파일-타임 정적값이어야 한다.
//        $primary: var(--color-primary) 사용 절대 불가.
//        CSS Custom Properties(--color-*)와 값을 수동으로 동기화한다.
//
// 색상 변경 시 두 곳을 함께 수정:
//   1. 이 파일의 $변수 값
//   2. scss/3-generic/_root.scss의 --color-* 값
//
// Bootstrap Sass 변수 → CSS Custom Properties 매핑:
//   $primary (#0d6efd)        ↔ --color-primary
//   $secondary (#6c757d)      ↔ --color-secondary
//   $font-size-base (1.6rem)  ↔ --font-size-base
// ─────────────────────────────────────────────────────
$font-size-base: 1.6rem;    // 16px (62.5% REM 트릭 대응) ↔ --font-size-base
$primary:    #0d6efd;        // ↔ --color-primary
$secondary:  #6c757d;        // ↔ --color-secondary

@import "bootstrap/scss/bootstrap";
```

**_variables.scss** 전체 내용으로 교체:

```scss
// ====================================================
// artpqUX Bootstrap 오버라이드 변수 — 문서화 원본
// ====================================================
//
// [중요] 실제 Bootstrap 오버라이드 효력 위치:
//   scss/3-generic/_vendor.scss 내 @import 스코프
//
// 이 파일은 팀이 "어떤 값을 오버라이드하는지" 확인하는
// 문서 역할을 한다. 값을 변경할 때는 반드시
// _vendor.scss도 동일하게 업데이트할 것.
//
// ── 토큰 매핑 주석 (TOKEN-05) ──────────────────────
// Bootstrap Sass 변수 → CSS Custom Properties 매핑:
//   $primary (#0d6efd)        ↔ --color-primary    (scss/3-generic/_root.scss)
//   $secondary (#6c757d)      ↔ --color-secondary  (scss/3-generic/_root.scss)
//   $font-size-base (1.6rem)  ↔ --font-size-base   (scss/3-generic/_root.scss)
//
// [중요] $primary: var(--color-primary) 사용 불가
//        Sass 컴파일-타임 변수에 CSS 런타임 함수(var()) 전달 불가.
//        Bootstrap tint-color(), shade-color() 등 색상 함수가 깨짐.
// ─────────────────────────────────────────────────────
//
// ====================================================

// ----------------------------------------------------
// 62.5% REM 트릭 대응
// html { font-size: 62.5% } → 1rem = 10px
// Bootstrap 기본 1rem = 16px 유지를 위해 1.6rem으로 재설정
// ----------------------------------------------------
$font-size-base: 1.6rem; // 16px ↔ --font-size-base

// ----------------------------------------------------
// 색상 오버라이드
// 값 변경 시 _vendor.scss + _root.scss도 동일하게 수정
// ----------------------------------------------------
$primary: #0d6efd;       // ↔ --color-primary
$secondary: #6c757d;     // ↔ --color-secondary

// ----------------------------------------------------
// 간격 오버라이드 예시
// $spacer는 Bootstrap 기본 1rem (= 10px in 62.5% 환경)
// 필요 시 $spacer: 1.6rem으로 16px 기준 유지
// ----------------------------------------------------
// $spacer: 1rem;

// ----------------------------------------------------
// 기타 오버라이드 예시
// ----------------------------------------------------
// $border-radius: 0.4rem;  // 4px (기본 0.375rem = 3.75px)
// $font-family-base: "Pretendard GOV", sans-serif;
```

```scss
// ====================================================
// artpqUX 퍼블리싱 가이드 — 메인 진입점
//
// 로드 순서 (ITCSS 기반):
//   1-settings  → 변수 (문서화 원본) — CSS 출력 없음
//   2-tools     → 믹스인 6종 — CSS 출력 없음
//   3-generic   → _root.scss (CSS Custom Properties 토큰)
//   project-overrides → 프로젝트별 :root 오버라이드 (cascade 순서 보장)
//   4-elements  → _base.scss (62.5% REM), _font.scss, _common.scss
//   5-objects   → 레이아웃 패턴
//   6-components → UI 컴포넌트
//   7-utilities → 유틸리티 클래스
//   Bootstrap   → @import 격리 (@use 이후 마지막 배치)
//
// Bootstrap 임포트 전략:
//   Bootstrap 5.3.x는 내부적으로 @import 사용.
//   3-generic/_vendor.scss에 @import로 격리하고,
//   나머지 레이어는 @use/@forward 기반으로 유지한다.
//   Sass 규칙상 @use는 @import보다 먼저 선언해야 하므로,
//   @use 계층을 모두 먼저 선언하고 @import를 그 뒤에 배치한다.
//   $font-size-base 등 Bootstrap 오버라이드 변수는
//   _vendor.scss 내부에서 @import 스코프로 선언한다.
//
// Bootstrap 오버라이드 변수 위치:
//   3-generic/_vendor.scss 내 @import 스코프에서 선언
//   1-settings/_variables.scss는 문서화 원본 역할만 수행
// ====================================================

// @use 계층 — Sass 규칙상 @import보다 먼저 위치해야 한다
// 1. 설정 (변수 문서화 원본)
@use '1-settings' as settings;

// 2. 도구 (믹스인 6종)
@use '2-tools' as tools;

// 3. Generic (CSS Custom Properties 토큰)
@use '3-generic' as generic;

// 3.5. 프로젝트별 토큰 오버라이드 (3-generic 이후 배치 → cascade 보장)
@use '1-settings/project-overrides';

// 4~7. 커스텀 스타일 레이어
@use '4-elements' as elements;
@use '5-objects' as objects;
@use '6-components' as components;
@use '7-utilities' as utilities;

// Bootstrap (@import 격리 — @use 이후에 위치)
// _vendor.scss 내부가 @import 기반이므로 @use 불가
// CSS 출력 순서: Bootstrap이 커스텀 레이어 뒤에 오지만,
// 클래스 선택자 기반이므로 요소 스타일과 충돌 없음
@import '3-generic/vendor';
```
