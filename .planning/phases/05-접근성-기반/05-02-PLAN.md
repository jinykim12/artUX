1. `docs/accessibility/keyboard.md` 작성 (per D-06, A11Y-07):
   - 제목: "# 키보드 탐색 가이드"
   - KWCAG 2.1 조항 번호 명시: 2.1.1 키보드 사용 보장, 2.1.2 초점 이동
   - 섹션 구성:
     - **탭 순서 원칙**: DOM 순서 = 탭 순서, tabindex="0" vs tabindex="-1" vs tabindex 양수(금지) 설명
     - **포커스 가능 요소**: a[href], button, input, select, textarea, [tabindex="0"] 목록
     - **포커스 트랩**: 모달/다이얼로그에서 포커스 순환 패턴 설명 (코드 구현은 Phase 9, 여기서는 원칙만)
     - **키보드 단축키**: Enter(링크/버튼 활성화), Space(체크박스/라디오), Escape(모달 닫기), 방향키(탭/메뉴 탐색)
     - **커스텀 위젯**: div/span에 role + tabindex + 키보드 이벤트 추가 필요성 안내
     - **체크리스트**: 마크다운 체크박스 형식 5~7개 항목
   - 본문건너뛰기 링크(skip-nav)와의 연관 관계 언급 (Plan 01에서 SCSS 구현)

2. `docs/accessibility/forms.md` 작성 (per D-06, A11Y-05 docs-only coverage — SCSS 폼 컴포넌트 구현은 Phase 7):
   - 제목: "# 폼 접근성 가이드"
   - KWCAG 2.1 조항 번호 명시: 3.4.1 레이블 제공, 3.4.2 오류 정정
   - 섹션 구성:
     - **label-input 연결**: for/id 매칭 패턴, aria-labelledby 대안, HTML 코드 예시
     - **필수 입력(required)**: required 속성 + aria-required="true" + 시각적 표시(*) 병행
     - **오류 메시지**: aria-describedby로 에러 메시지 연결, aria-invalid="true" 상태 표시, HTML 코드 예시
     - **fieldset/legend**: 라디오/체크박스 그룹핑 패턴
     - **placeholder 주의**: placeholder는 label 대체 불가, 보조 힌트로만 사용
     - **체크리스트**: 마크다운 체크박스 형식 5~7개 항목
   - 참고: 폼 SCSS 컴포넌트 코드는 Phase 7에서 구현, 이 문서는 마크업 패턴 가이드
