`docs/accessibility/dynamic-content.md` 작성 (per D-06, D-16, A11Y-09):

- 제목: "# 동적 콘텐츠 접근성 패턴"
- 안내문: "이 문서는 패턴 가이드입니다. SCSS 코드 구현은 Phase 9(오버레이 컴포넌트)에서 진행됩니다."
- KWCAG 2.1 관련 조항: 4.2.1 웹 애플리케이션 접근성 준수

섹션 구성:

1. **aria-live 영역**:
   - `aria-live="polite"` vs `aria-live="assertive"` 차이점
   - 사용 예: 검색 결과 갱신, 알림 메시지, 로딩 상태
   - HTML 코드 예시:
     ```html
     <div aria-live="polite" aria-atomic="true">
       검색 결과 {n}건이 표시됩니다.
     </div>
     ```

2. **role="alert"**:
   - `role="alert"` = 암시적 `aria-live="assertive"`
   - 사용 예: 폼 제출 오류, 시스템 경고
   - HTML 코드 예시

3. **모달 포커스 트랩**:
   - 원칙: 열릴 때 모달 내부로 포커스 이동, Tab 순환, Escape로 닫기
   - 닫힐 때 트리거 요소로 포커스 복원
   - `aria-modal="true"`, `role="dialog"`, `aria-labelledby` 패턴
   - HTML 구조 예시 (JS 동작은 설명만, 코드 구현은 Phase 9)

4. **탭 패널**:
   - `role="tablist"`, `role="tab"`, `role="tabpanel"`
   - `aria-selected="true"`, 방향키 탐색
   - HTML 구조 예시

5. **아코디언/토글**:
   - `aria-expanded="true/false"`, `aria-controls`
   - HTML 구조 예시

6. **체크리스트**: 동적 콘텐츠 접근성 점검 항목 5~7개 (마크다운 체크박스)
