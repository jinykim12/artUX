    5종 마크업 섹션 (per D-01, 07-RESEARCH.md Pattern 2 기반):

    a) ## 텍스트 입력 (input)
       - 기본 텍스트 입력: label for/id 매칭, aria-required="true" + 시각적 "*" + sr-only "(필수)", autocomplete
       - 오류 상태 입력: is-invalid 클래스 + aria-invalid="true" + aria-describedby + role="alert" 오류 메시지

    b) ## 선택 (select)
       - 기본 select: label for/id 매칭, 기본 옵션 "선택하세요"

    c) ## 텍스트 영역 (textarea)
       - 기본 textarea: label for/id 매칭, rows, aria-describedby로 힌트 텍스트 연결

    d) ## 체크박스 (checkbox)
       - 단일 체크박스: form-check 구조, for/id 매칭
       - 체크박스 그룹: fieldset/legend 필수 패턴

    e) ## 라디오 (radio)
       - 라디오 그룹: fieldset/legend 필수 패턴

    f) ## 오류 처리 (JS 예시) (per D-09, D-10)
       - IIFE + querySelector 패턴 (Phase 6 GNB JS 패턴과 동일)
       - submit 이벤트 가로채기
       - [data-validate] 필드 순회
       - classList.add('is-invalid') + setAttribute('aria-invalid', 'true') 쌍으로 처리
       - role="alert" + style.display 토글
       - 첫 번째 오류 필드로 focus() 이동
       - 오류 해제 시 classList.remove + removeAttribute 쌍으로 처리

    g) ## 접근성 체크리스트 (간략)
       - label-input 연결 확인
       - error 메시지 aria-describedby 연결 확인
       - required 표시 (aria-required + 시각적) 확인
       - checkbox/radio 그룹 fieldset/legend 확인
       - `docs/accessibility/forms.md` 전체 가이드 참조 안내
