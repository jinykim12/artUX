**button.md 작성 (per D-01, D-02, D-03, COMP-02):**

Phase 7 forms.md 패턴을 따른다. 상단에 컴포넌트명, KWCAG 기준, 적용 수준, 작성일, 참고 문구(SCSS 파일 매칭) 작성.

포함 섹션:

1. **기본 버튼 (Primary)**
```html
<!-- 기본 버튼 — type="button" 명시 (form 내 submit 방지) -->
<button type="button" class="btn btn-primary">확인</button>
```

2. **Secondary / Outline / Ghost 버튼**
```html
<button type="button" class="btn btn-secondary">취소</button>
<button type="button" class="btn btn-outline-primary">더보기</button>
<button type="button" class="btn btn-ghost">닫기</button>
```

3. **비활성 버튼**
```html
<!-- disabled 속성 + aria-disabled="true" — 스크린 리더에서도 비활성 안내 -->
<button type="button" class="btn btn-primary" disabled aria-disabled="true">
  제출 불가
</button>
```

4. **아이콘 버튼**
```html
<!-- 아이콘 전용 버튼 — aria-label 필수 (텍스트 없으므로) -->
<button type="button" class="btn btn-ghost btn--icon" aria-label="삭제">
  <svg aria-hidden="true" ...>...</svg>
</button>
```

5. **링크 스타일 버튼 (a 태그)**
```html
<!-- a 태그를 버튼처럼 사용할 때 — role="button" 추가 -->
<a href="#" class="btn btn-primary" role="button">다운로드</a>
```

6. **접근성 체크리스트** 섹션:
   - type="button" / type="submit" 명시
   - disabled 시 aria-disabled="true" 병행
   - 아이콘 전용 시 aria-label 필수
   - a 태그 사용 시 role="button" 추가
   - 로딩 상태 시 aria-busy="true" + 텍스트 변경 안내

**card.md 작성 (per D-01, D-02, D-03, COMP-04):**

포함 섹션:

1. **기본 카드**
```html
<div class="card">
  <div class="card-body">
    <h3 class="card-title">카드 제목</h3>
    <p class="card-text">카드 설명 텍스트입니다.</p>
  </div>
</div>
```

2. **이미지 카드**
```html
<!-- 이미지 카드 — alt 속성에 이미지 설명 필수 (장식 이미지면 alt="") -->
<div class="card">
  <img src="example.jpg" class="card-img-top" alt="프로젝트 미리보기 화면">
  <div class="card-body">
    <h3 class="card-title">프로젝트명</h3>
    <p class="card-text">프로젝트 설명입니다.</p>
  </div>
</div>
```

3. **링크 카드 (전체 클릭)**
```html
<!-- 링크 카드 — article 태그 사용, 내부 a 태그가 카드 전체를 감싸지 않도록 주의 -->
<article class="card">
  <div class="card-body">
    <h3 class="card-title">
      <a href="/detail/1">카드 제목</a>
    </h3>
    <p class="card-text">설명 텍스트</p>
  </div>
</article>
```

4. **헤더/푸터 카드**
```html
<div class="card">
  <div class="card-header">카테고리</div>
  <div class="card-body">
    <h3 class="card-title">제목</h3>
    <p class="card-text">내용</p>
  </div>
  <div class="card-footer">
    <button type="button" class="btn btn-primary">자세히</button>
  </div>
</div>
```

5. **접근성 체크리스트** 섹션:
   - 이미지에 적절한 alt 속성 필수 (장식 이미지 alt="", 의미 이미지 설명)
   - 카드 제목에 적절한 heading 레벨 사용 (h2~h4 페이지 구조에 맞게)
   - 링크 카드에서 a 태그 텍스트가 카드 목적을 설명해야 함
