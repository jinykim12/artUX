    # 컨벤션
    소개: 아트피큐 팀 퍼블리싱 코딩 컨벤션. 신규 프로젝트 시작 시 이 규칙을 기준으로 한다.

    ## SCSS 네이밍 규칙
    BEM 변형 (팀 표준) — 코드 예시:
      .card (블록), .card__title (요소, 언더스코어 2개), .card--featured (수정자, 하이픈 2개)
    파일 네이밍: 소문자 + 하이픈, 언더스코어 접두사 (_button.scss), ITCSS 레이어 번호 (6-components/)
    클래스 원칙: Bootstrap 클래스는 추가만, js-* 접두사는 스타일 부여 금지

    ## 코딩 스타일
    들여쓰기: 2 spaces (탭 금지), 인코딩: UTF-8, 줄 끝: LF
    SCSS 작성 순서 5그룹: 포지셔닝, 박스 모델, 타이포그래피, 시각 효과, 기타
    접근성 필수 규칙 3항:
      - transition: none 원칙 (공공기관 납품 시 전체 제거)
      - 포커스 아웃라인 제거 금지 (:focus-visible 항상 유지)
      - aria-* 속성 + CSS 클래스 동시 적용

    ## Git 커밋 규칙
    커밋 메시지 형식: type(scope): 한국어 설명
    타입 테이블: feat, fix, docs, style, refactor, chore + 용도 설명
    브랜치 전략: main(납품 가능), develop(개발 통합), feature/이슈번호-설명

    ## HTML 마크업 규칙
    시맨틱 태그 우선, alt 속성 필수 (장식용 alt=""), type="button" 명시, lang="ko" 필수

  - 접근성: aria-label="코드 복사", 복사 성공 시 버튼 텍스트/aria-label 변경

구현 내용 (IIFE 패턴, 'use strict'):

  function initCopyButtons():
    - document.querySelectorAll('pre > code') 로 모든 코드 블록 선택
    - 각 코드 블록의 부모 pre에 .copy-btn 없으면 버튼 생성
    - pre.style.position = 'relative'
    - 버튼 생성:
        className = 'copy-btn'
        type = 'button'
        aria-label = '코드 복사'
        textContent = '복사'
        인라인 스타일: position:absolute, top:.5rem, right:.5rem, padding:.25rem .6rem,
          font-size:.75rem, background:#fff, border:1px solid #dee2e6,
          border-radius:.25rem, cursor:pointer, line-height:1.4, z-index:1
    - 클릭 이벤트:
        navigator.clipboard.writeText(code.textContent).then() 성공 시:
          btn.textContent = '복사됨!'
          btn.setAttribute('aria-label', '코드가 복사되었습니다')
          setTimeout 2000ms 후 원복 (textContent = '복사', aria-label = '코드 복사')
        .catch() 실패 시:
          btn.textContent = '실패'
          setTimeout 2000ms 후 원복
    - pre.appendChild(btn)

  실행:
    document.readyState === 'loading' 이면 DOMContentLoaded 이벤트 후 실행
