    1. BEM 변형 (팀 표준) — 블록/요소/수정자 예시 코드 포함
       - 블록: `.card`, `.form`, `.modal`
       - 요소: `.card__title`, `.card__body` (언더스코어 2개)
       - 수정자: `.card--featured`, `.btn--ghost` (하이픈 2개)

    2. Bootstrap 유틸리티 사용 기준
       - Bootstrap이 제공하는 유틸리티 클래스(`d-flex`, `mt-3`, `text-center` 등)는 그대로 사용
       - 기존 Bootstrap 클래스를 직접 수정하지 않는다 — CSS 선택자 오버라이드 방식 사용
       - Bootstrap 유틸리티로 해결되지 않는 경우에만 커스텀 클래스 작성

    3. 커스텀 클래스 작성 기준
       - HTML 마크업에서 커스텀 클래스는 마지막에 위치: `class="d-flex align-items-center card__header"`
       - `.btn-ghost` 사례처럼 Bootstrap에 없는 variant는 팀 표준으로 명시

    4. js- 접두사 규칙
       - `js-` 접두사 클래스는 JS 훅 전용 — 스타일 부여 절대 금지
       - 예: `<button class="btn btn-primary js-modal-trigger">` — `js-modal-trigger`에는 CSS 작성 안 함

    5. 파일 네이밍 규칙
       - SCSS partial: 소문자 + 하이픈, 언더스코어 접두사: `_button-group.scss`
       - ITCSS 레이어 번호 디렉토리: `6-components/`, `2-tools/`
       - HTML 파일: 소문자 + 하이픈: `main-visual.html`

    6. 피해야 할 패턴 (Bad/Good 대조 예시 포함)
       - camelCase 클래스명 사용 금지
       - 의미 없는 약어 (`wrp`, `cont`) 지양 — `wrapper`, `container` 사용
       - Bootstrap 클래스에 직접 스타일 덮어쓰기 금지

    파일 첫 줄: YAML frontmatter 블록 (삼중 대시로 감싼 YAML 헤더 포함):
      layout: layouts/base.njk
      title: SCSS 작성 규칙
      tags: conventions
      section: conventions
      permalink: /conventions/scss-rules/

    문서 구성 (모두 포함):

    1. 중첩 깊이 제한 (3단계)
       - 최대 3단계: `.parent > .child > .grandchild` 까지만 허용
       - Bad/Good 대조 코드 예시 포함
       - 이유: 과도한 중첩은 CSS 명시도 증가 및 유지보수 어려움

    2. 속성 순서 (5그룹) — 기존 index.md 내용 그대로 이관 + 확장
       - 그룹 1: 포지셔닝 (position, top/right/bottom/left, z-index)
       - 그룹 2: 박스 모델 (display, width/height, padding, margin, overflow)
       - 그룹 3: 타이포그래피 (font-family, font-size, font-weight, line-height, color, text-align)
       - 그룹 4: 시각 효과 (background, border, border-radius, box-shadow, opacity, transition)
       - 그룹 5: 기타 (cursor, pointer-events, user-select)
       - 실제 .component {} 블록 예시 코드 포함

    3. @use vs @import 전략
       - 팀 표준: `@use` 방식 (Dart Sass 1.98.0+ 권장)
       - Bootstrap은 `scss/3-generic/_vendor.scss` 격리 전략으로 `@import` 유지 (Bootstrap이 @use 미지원)
       - 팀 파일 간 참조: `@use '../2-tools/mixin' as *;`
       - 기존 `@import` 프로젝트와의 공존: `@import` → `@use` 마이그레이션은 신규 파일부터 점진적 전환

    4. 주석 규칙
       - 섹션 구분: `// ============================================================`
       - 파일 상단: 파일 목적 한 줄 주석 `// [파일명] — 역할 설명`
       - TODO: `// TODO: 내용` (팀원 확인 필요 시)
       - 납품 전 제거 대상: `// TEMP:` 또는 `// DEBUG:` 접두사

    5. 공공기관 납품 필수 규칙
       - `transition: none` — 납품 시 전체 제거 (Phase 09 결정 상속)
       - `animation: none` — 동일 원칙 적용
       - CSS Custom Properties(`var(--)`) 우선 사용 — 하드코딩 컬러값 금지

    6. @forward 사용 지침
       - `scss/style.scss`에서 각 컴포넌트 partial을 `@forward`로 집계
       - race condition 방지: 컴포넌트 추가 시 style.scss @forward 동시 등록
  </action>
